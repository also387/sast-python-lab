# Static Application Security Testing (SAST)

Laboratory work on code security analysis and vulnerability classification.

## Overview
This repository contains practical assignments focused on static code analysis using the **Bandit** scanner. The project demonstrates the complete process of identifying, classifying (triage), and remediating vulnerabilities in Python-based web applications (Flask).

## Technologies & Standards
* **Language / Framework:** Python, Flask
* **Security Tool:** Bandit
* **Standards:** CWE, OWASP Top 10

## Project Structure
* `vulnerable_app.py` / `lab2_app.py` — Initial applications containing intentional security flaws.
* `vulnerable_app_fixed.py` / `lab2_app_fixed.py` — Remediated versions of the code with security patches applied.
* `*report.json` — Raw analysis reports generated by Bandit.

## Addressed Vulnerabilities
The following common weaknesses were analyzed and fixed during the work:
* **CWE-89** (SQL Injection)
* **CWE-798** (Hardcoded Secrets)
* **CWE-78** (OS Command Injection)
* **CWE-79** (Cross-Site Scripting)
* **CWE-502** (Insecure Deserialization)
* **CWE-400** (Uncontrolled Resource Consumption / DoS)
